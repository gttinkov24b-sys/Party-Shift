# Party Shift

Party Shift is a narrative-driven, console-based simulation game developed in C# using Visual Studio. This project was created as part of a passion project and a school assignment, focusing on creating an immersive experience through logic, timing, and text-based interaction. Throughout the development process, I found a deep passion for game design and the challenge of building a functional world within a Command Line Interface (CLI).

### Technical Skills and Concepts Learned:

* **Process Control and Application Exit:** I discovered how to forcefully and gracefully terminate a program using **Environment.Exit(0)**. This was crucial for handling "Game Over" states, such as when the player gets fired or arrested, ensuring the application closes immediately rather than continuing into the next block of code.

* **UI/UX Design in the Console:** One of the biggest challenges was making a text-only game look professional. I learned how to create a custom algorithm to calculate the center of the console window. By subtracting the length of a string from the total window width and dividing by two, I was able to create centered headers and menus that adapt to the user's screen.

* **Loop Logic (For vs Foreach):** I gained a clear understanding of when to use specific loops. I utilized **for** loops when I needed a specific number of iterations (like the 5-question limit in the police interrogation) and **foreach** loops when I needed to scan through a collection of unknown size (like listing the specific items the player chose to buy in their inventory).

* **Visual Feedback with Colors:** I learned how to manipulate **Console.ForegroundColor** to provide instant feedback to the player. Using green for success, red for failure, and custom colors for different job themes helped make the game more intuitive and visually engaging without needing high-end graphics.

* **Data Management (Lists vs Arrays):** I mastered the distinction between fixed and dynamic data. I used **Arrays** for static elements like the job titles and drink menus that do not change during gameplay. In contrast, I used **Lists** for the player's shopping bag, allowing the game to add or remove items dynamically as the player made choices.

### Project Reflection:

Building Party Shift taught me that programming is as much about the player's experience as it is about the code. Managing the game's economy, balancing the difficulty of the "Park Patrol" mini-game, and ensuring the "Secret Ending" felt rewarding were the highlights of this creation process. This project has solidified my interest in software development and the creative potential of C#.
